# 5. Relación de la calidad del dato con la calidad del software según UNE 0081

En una aplicación como NutriTracker, la calidad del dato tiene un impacto directo en la calidad del software. La aplicación no solo almacena información, sino que utiliza esos datos para calcular calorías, macronutrientes, objetivos diarios, evolución del peso y progreso del usuario. Por eso, si los datos son incorrectos, incompletos o están mal documentados, el funcionamiento de la aplicación también se ve afectado.

A continuación se analiza cómo la calidad del dato influye en varios atributos de calidad del software según la UNE 0081, aplicándolo al caso concreto de NutriTracker.

| Atributo | Relación con la calidad del dato | Ejemplo en NutriTracker | Problema potencial |
|---|---|---|---|
| Fiabilidad | La aplicación depende de que los datos nutricionales sean correctos para mostrar resultados fiables. Si los valores de calorías o macronutrientes son erróneos, los cálculos diarios dejan de ser válidos. | Si un producto tiene registradas 9000 kcal por 100 g por un error de importación, el resumen diario del usuario mostrará un consumo calórico completamente incorrecto. | El usuario puede pensar que ha superado su objetivo diario cuando en realidad no es así. Esto reduce la confianza en la aplicación. |
| Usabilidad | Cuando los datos están duplicados, incompletos o escritos de forma poco clara, al usuario le cuesta más usar la aplicación correctamente. | Si aparecen varios alimentos muy parecidos, como “Yogur natural”, “yogur Natural” y “Yogurt natural”, el usuario no sabe cuál seleccionar. | La experiencia de uso empeora y el usuario puede registrar mal sus comidas o dejar de usar la aplicación. |
| Eficiencia | Los datos duplicados o mal estructurados pueden hacer que las búsquedas, filtros y cálculos sean más lentos. | Si la tabla de productos acumula muchos alimentos repetidos importados desde fuentes externas, la búsqueda de alimentos puede volverse más lenta. | La aplicación tarda más en responder, especialmente en búsquedas o resúmenes diarios con muchos registros. |
| Seguridad | La aplicación maneja datos personales y datos relacionados con la salud, por lo que deben estar bien identificados y protegidos. | Datos como el email, peso, altura, objetivo corporal, IMC o fotos de progreso deben considerarse sensibles. | Si estos datos no están marcados como sensibles o no tienen controles adecuados, puede haber exposición de información personal y problemas de cumplimiento normativo. |
| Mantenibilidad | Si los datos no están bien documentados, es más difícil modificar o ampliar la aplicación sin cometer errores. | Si un campo como `goal_type` o `activity_level` no tiene una descripción clara, un desarrollador nuevo puede no entender qué valores acepta o cómo se usa. | Se pueden introducir errores al añadir nuevas funciones, cambiar cálculos o modificar el modelo de datos. |
